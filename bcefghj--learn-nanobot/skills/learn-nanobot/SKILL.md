# 日报生成技能

## 使用方法
当用户请求生成日报时，按照以下流程引导并生成。

## 信息收集
如果用户未提供完整信息，依次询问：
1. 今天完成了哪些工作？
2. 遇到了什么问题或阻碍？
3. 明天计划做什么？

## 输出格式

```
📋 工作日报 - {日期}

👤 {姓名/部门}（如用户提供）

━━━ 今日完成 ━━━
1. {完成事项1}
2. {完成事项2}
...

━━━ 遇到问题 ━━━
1. {问题描述}
   → 当前进展/解决方案
...
（如无问题则写"无"）

━━━ 明日计划 ━━━
1. {计划事项1}
2. {计划事项2}
...

━━━ 备注 ━━━
{其他需要说明的内容}
（如无则省略此节）
```

## 生成规则
- 语言简练，每条不超过两句话
- 完成事项标注关键成果（如数据指标、版本号等）
- 问题要包含影响范围和当前进展
- 计划事项按优先级排序
- 如果用户只提供了简短描述，帮助扩展为完整的日报条目
