# `core-processor` Generation Guide

This module is the shared compile-time engine for Micronaut injection. Language-specific processors provide Micronaut AST elements; `core-processor` turns those elements and annotation metadata into generated bytecode models for bean definitions, executable methods, bean introspections, AOP proxies, expression classes, and configuration metadata.

Use this file when changing `core-processor/src/main/java/**` so generated bytecode, runtime interfaces in `inject`, and annotation processing behavior stay aligned.

## Mental Model

The normal compile-time path is:

1. A language module such as `inject-java` supplies `ClassElement`, `MethodElement`, `FieldElement`, `PropertyElement`, and `ParameterElement` implementations.
2. `TypeElementVisitor` implementations inspect and mutate annotation metadata. Examples include `ConfigurationMetadataWriterVisitor`, `ConfigurationReaderVisitor`, `ExecutableVisitor`, and `IntrospectedTypeElementVisitor`.
3. `BeanDefinitionCreatorFactory` chooses the bean creation strategy for each `ClassElement`.
4. Creator classes call `ElementBeanDefinitionBuilder` and `ElementProxyBuilder` APIs.
5. Writers such as `BeanDefinitionWriter`, `ExecutableMethodsDefinitionWriter`, `BeanIntrospectionWriter`, `AopProxyWriter`, `RuntimeProxyBeanDefinitionWriter`, and `EvaluatedExpressionWriter` build `io.micronaut.sourcegen.model.ObjectDef` instances.
6. The language processor writes bytecode through `ByteCodeWriterUtils` and registers service descriptors when an `OutputObjectDef` has a service type.

Do not think of this module as producing Java source. Most writers build sourcegen AST objects such as `ClassDef`, `MethodDef`, `ExpressionDef`, `StatementDef`, and `TypeDef`; `ByteCodeWriterUtils` then emits bytecode with ASM.

## AST Layer

`io.micronaut.inject.ast` is a language-neutral AST over Java, Kotlin, and Groovy. It is the boundary between language-specific symbol APIs and shared generation logic.

Important AST types:

| Type | Role |
| --- | --- |
| `Element` | Common annotated, named, documented, mutable metadata node. It can represent a class, method, field, parameter, package, enum constant, or property-like element. |
| `ClassElement` | Type model with assignability, generics, optional/container helpers, constructors, bean properties, fields, methods, interfaces, super types, records, enums, arrays, and proxy detection. |
| `MethodElement` | Method or constructor model. `getMethodAnnotationMetadata()` is method-only metadata; `getAnnotationMetadata()` may combine class and method metadata except for constructors. |
| `ParameterElement` | Method or constructor argument model used for injection points, executable methods, constructor values, and Kotlin default detection. |
| `FieldElement` | Field model used for field injection, factory field beans, and field-backed bean properties. |
| `PropertyElement` | Synthesized bean property. Configuration readers process these first because the property is the user-facing binding unit. |
| `KotlinParameterElement` | Marks a Kotlin parameter that has a default value. Writers use it to call Kotlin synthetic default constructors and methods. |
| `ElementQuery` and `PropertyElementQuery` | The preferred way to select enclosed elements. Use queries instead of ad hoc scans when possible. |

AST elements are often copied with modified metadata using `withAnnotationMetadata(...)`, `withParameters(...)`, or direct `annotate(...)` calls. This is intentional: visitors and creators add synthetic annotations such as `@Property`, `@ConfigurationReader`, interceptor bindings, validation markers, and `@Introspected` before code generation.

Be careful with metadata layers. `AnnotationMetadataHierarchy` is frequently used to combine type metadata with method, factory, property, or parameter metadata while preserving declared-vs-inherited lookup rules. Generated code often stores an `AnnotationMetadataReference` back to a generated bean definition to avoid duplicating metadata.

## Generated Output

The major generated artifacts are:

| Writer | Generated class shape | Service descriptor |
| --- | --- | --- |
| `BeanDefinitionWriter` | `$Bean$Definition` classes extending `AbstractInitializableBeanDefinitionAndReference` unless a specialized super type is used. | `BeanDefinitionReference` |
| `ExecutableMethodsDefinitionWriter` | `$Bean$Definition$Exec` classes extending `AbstractExecutableMethodsDefinition`. | None |
| `BeanIntrospectionWriter` | `$Bean$Introspection` or `$fully_qualified_Bean$Introspection` classes extending `AbstractInitializableBeanIntrospectionAndReference` or enum equivalent. | `BeanIntrospectionReference` |
| `AopProxyWriter` | A proxy type named from the target definition plus `$Intercepted`, and a proxy bean definition for that proxy type. | Proxy bean definition is a `BeanDefinitionReference` |
