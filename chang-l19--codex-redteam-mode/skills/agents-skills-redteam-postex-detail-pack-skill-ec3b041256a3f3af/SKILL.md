# redteam-postex-detail-pack

## Purpose
Use this pack when hook routing has already narrowed the task into this technical domain and you want concrete, high-signal downstream skills instead of abstract methodology.

## Use pattern
1. Keep the current phase / router / selected path compact.
2. Open only the matching leaf skills for the visible bug class or operator task.
3. Pull one reference checklist from `references/` if you need more detail.
4. Avoid loading unrelated skills from the same domain.
