# The Omega 科研宪章

## Core Principles

### I. 第一性原理优先

- 每个结论必须有可追溯的推导链：定义/公理/已证引理 → 命题/定理 → 推论。
- 任何“物理解释/直觉”不得替代证明；若仅为启发，必须标注为解释性叙述，
  且不得作为后续推理前提。
- 任何引用外部结果时，必须明确其被使用的精确形式（定理陈述、条件、常数/界），
  并在需要处给出可复核的引用位置。

理由：顶级期刊审稿的核心是可验证的逻辑闭合；第一性原理是最小输入的起点。

### II. 最小输入原则（最少公理、最少假设、最少外部输入）

- 新增公理/假设/外部输入前，必须给出必要性说明：若移除该输入，将导致哪一步无法完成，
  并指出最小失败点。
- 对同一目标，优先选择输入更少、依赖链更短、可审计代价更低的表述；
  禁止为了叙事完整而引入不可检验的额外结构。
- 对经验性拟合或数值参数，必须区分定义参数、可拟合参数、可预测参数，并在文稿中明确标注。

理由：输入最小化可降低自由度，提升可证伪性与可累积性。

### III. 公理与假设分层标注（显式依赖链）

- 文稿中出现的每一条基础陈述，必须归类为：定义、约定、公理、假设、猜想、推导结果、
  经验事实之一。
- 任何跨层跳跃（例如：从数学对象到物理指称）必须单独列出桥接假设，并明确其可检验/
  可反驳路径或局限。
- 禁止隐含公理：若某步推理需要额外条件，必须显式写出并进入依赖链。

理由：分层与依赖链是同行评审的可读性基础，也是长期维护大型理论体系的可扩展方式。

### IV. 可证伪与量化约束（含误差预算）

- 任何经验性、数值性或可观测主张必须配套：观测量定义、数据来源/生成方式、误差指标、
  误差预算或上界。
- 纯理论结果必须给出至少一种可审计的一致性检查：边界情形、极限行为、已知特例复现、
  维度/单位一致性等。
- 负面结果与适用边界必须被记录；禁止以未来工作替代已知限制。

理由：顶级期刊要求可检验的新增信息，量化约束与误差预算是可检验性的具体化。

### V. 可复现、可审计、可再生成（非可选）

- 所有表格、图像、数值结果必须由脚本生成并可一键再现；
  禁止手工修改生成后的 `.tex` 内容或直接嵌入数据。
- 生成脚本必须记录：输入数据/参数、随机种子（如有）、版本信息与运行方式；
  输出必须可追溯到源脚本。
- 若结果依赖外部服务或数据库，必须提供可复现的导出/快照策略或等价的最小可复现数据集。

理由：可复现性是科研可信度的底线，也是长期迭代与审计的基础设施。

## 论文写作与结构标准（顶级期刊）

- **语言**：本仓库规范性文档默认中文；英文版文档以 `_en.md` 结尾（LaTeX 同理使用 `_en.tex`）。
- **结构要求**：每篇论文必须包含（按需要合并但不得缺失其功能）：
  - 摘要与贡献点（最小主张清单）
  - 引言（问题、动机、与现有工作差异）
  - 预备知识/定义/公理与假设（依赖链）
  - 主要结果（定理/命题/算法）与证明/推导
  - 验证路径（数值、实验、或一致性检查）与误差说明
  - 讨论、局限与可证伪点
  - 可复现性声明（代码/脚本/数据路径）
- **写作纪律**：
  - 不用权威背书替代论证；直觉只能作为解释性补充。
  - 不扩大叙事范围：只陈述被本文证明或明确假设的内容。
  - 禁止 “口径化”, 需要带明确引理/定理与引用定位
  
## 科研工作流、复现与质量门禁

- **不确定性处理**：遇到难以处理或不确定的问题，或者希望询问时, 必须先检索 `docs/papers/` 内是否已有处理与约定。
- **数据与脚本管线**：
  - 任何写入论文的数值/图表必须来自脚本；脚本输出直接进入 `.tex` 或其输入文件。
  - 长时间运行（循环）的脚本必须每分钟输出一次进度信息。

- **合规门禁（提交/合并前）**：
  - 检查输入清单：新增公理/假设/外部输入是否必要且显式？
  - 检查依赖链：关键结论是否可追溯到明确的基础层？
  - 检查复现性：结果是否可一键再生成且参数/版本齐备？
  - 检查边界与局限：适用范围与反例/失败模式是否记录？
