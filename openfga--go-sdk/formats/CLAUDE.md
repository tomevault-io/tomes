## Code Generation

**Parts of this SDK are auto-generated.** The following files are generated by [OpenAPI Generator](https://openapi-generator.tech) from the [sdk-generator](https://github.com/openfga/sdk-generator) repo and **will be wiped on every regeneration**:

**Generated Files:**
- All `model_*.go` files (data model files)
- `api_open_fga.go` (low-level API client, deprecated)
- `streaming.go` (streaming response handler)
- `internal/constants/constants.go` (SDK constants)
- `docs/**` (~90 API reference markdown files)

**How to identify generated files:** Look for the header comment containing `NOTE: This file was auto generated by OpenAPI Generator ... DO NOT EDIT.`

**If you need to change generated files:** Contributors may modify generated files in this repo, but changes must be sourced from sdk-generator. Submit changes to https://github.com/openfga/sdk-generator first and link the sdk-generator PR in this repo's PR description. Unlinked changes will be overwritten on the next regeneration.

**Preferred pattern for new API methods:** Use `api_executor.go` (hand-written) rather than `api_open_fga.go` (generated). The executor pattern is newer and more maintainable.

## Architecture

### Dual-Layer Design

1. **Low-Level Layer** (`APIClient`, `APIExecutor`)
   - HTTP request marshaling, retry logic, error handling
   - Direct API responses (raw structs)
   - Use when you need fine-grained control

2. **High-Level Layer** (`OpenFgaClient` in `client/`)
   - Batch request splitting (50 tuples per batch max)
   - Parallel request execution (10 max parallel requests)
   - Credential injection and token refresh
   - Telemetry instrumentation
   - Use for standard operations (Check, BatchCheck, Expand, etc.)

### Package Purposes

- **Root package** (`openfga`): Core types, configuration, APIClient
- **`client/`**: High-level SDK wrapper, method implementations
- **`credentials/`**: Credential types and injection
- **`oauth2/`**: OAuth2 token flows and Bearer token transport
- **`telemetry/`**: OpenTelemetry metric recording
- **`internal/constants/`**: Batch limits (50 tuples), parallel request limits (10)
- **`internal/utils/retryutils/`**: Exponential backoff, Retry-After header parsing

## Key Patterns

### Credentials

Three credential types; configure via `ClientConfiguration`:
- **None** (default): No authentication
- **API Token**: Static bearer token via `Authorization` header
- **Client Credentials**: OAuth2 flow with automatic token refresh

```go
config := &client.ClientConfiguration{
    ApiUrl: "https://api.fga.example",
    Credentials: &credentials.Credentials{
        Method: credentials.CredentialsMethodClientCredentials,
        Config: &credentials.Config{
            ClientCredentialsApiTokenIssuer: "https://issuer.fga.example",
            ClientCredentialsClientId:       "client-id",
            ClientCredentialsClientSecret:   "client-secret",
        },
    },
}
```

### Streaming (StreamingChannel)

For APIs returning NDJSON streams (e.g., `StreamedListObjects`):

```go
channel, err := client.StreamedListObjects(ctx, storeId, request)
for {
    select {
    case result := <-channel.Results:
        // Process result
    case err := <-channel.Errors:
        // Handle error; loop terminates
        return err
    }
}
channel.Close()
```

**Critical:** Always drain both `Results` and `Errors` channels to prevent goroutine leaks.

### Retry Strategy

Automatic exponential backoff for retryable errors:
- **HTTP 429** (rate limit): Always retried
- **HTTP 5xx** (server error): Retried with backoff
- **HTTP 4xx** (client error): Never retried
- **Retry-After header:** Respected and capped at 1800 seconds

Custom retry params via `Configuration.RetryParams`.

### Telemetry (OpenTelemetry)

Metrics emitted for all client operations:
- `fga_client_credentials_request` (counter)
- `fga_client_request_duration` (histogram, ms)
- `fga_client_query_duration` (histogram, ms)
- `fga_client_http_request_duration` (histogram, ms)

**For unit tests:** Use `noop.NewMeterProvider()` to suppress metric export, not live OTel exporters.

## Gotchas

1. **Generated files get wiped.** Any edits to `model_*.go`, `api_open_fga.go`, `streaming.go`, `internal/constants/constants.go`, or `docs/*` will be lost on regeneration. Contributors may modify them, but must submit changes to [sdk-generator](https://github.com/openfga/sdk-generator) and link the PR.

2. **Token expiry is not exact.** OAuth2 tokens refresh 5тАУ10 minutes early due to:
   - 300s threshold before expiry
   - 300s random jitter
   - Do not write time-based assertions in tests; mock token provider instead.

3. **StreamingChannel goroutine leak.** Channels must be fully drained before returning:
   ```go
   // WRONG: leaves goroutine running
   for result := range channel.Results { ... }

   // RIGHT: drain both channels
   for {
       select {
       case result := <-channel.Results:
       case err := <-channel.Errors: return err
       }
   }
   ```

4. **golangci-lint excludes generated code.** Model files are intentionally skipped by `.golangci.yaml` so lint errors in generated code don't fail CI. Do not remove this exclusion.

5. **Batch limits are constants.** `BatchCheck` splits into max 50-tuple batches with 10 parallel requests. Both are defined in `internal/constants/constants.go`:
   - `ClientMaxBatchSize = 50`
   - `ClientMaxMethodParallelRequests = 10`

6. **Deprecated config fields.** `ApiScheme` and `ApiHost` are deprecated; use `ApiUrl` instead. New code should set only `ApiUrl`.

7. **Example submodules are independent.** Each `example/*/go.mod` is a separate project with its own dependencies. They are not tested in CI.
