# Skill: Findings to GitHub Issues

## Quando usar
Quando você já tem achados de análise e precisa transformá-los em:
- Issues
- Comentários de PR

## Escopo da skill
Inclui:
- Formatação Markdown
- Labels
- Priorização

Não inclui:
- Execução de APIs (autenticação)

## Entradas esperadas
- Lista de achados
- Prioridades e labels desejados

## Como analisar
1) Receber achados normalizados.
2) Transformar em texto Markdown.
3) Categorizar por labels.
4) Opcionalmente agrupar por tipo.

## Saída esperada
Markdown pronto para GitHub Issues e comentários

## Regras obrigatórias
- Título claro
- Evidência resumida
- Recomendações
- Labels por categoria

## Limitações
- Não executa operação de fato
