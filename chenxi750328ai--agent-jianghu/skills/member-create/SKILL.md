# 生成新成员（配套基础设施）

## 何时使用

- 需要新增一名项目成员（江湖级），并为其配备与陈正落/陈正霞/陈正与同套的「身份、恢复记忆、私有 RAG、职责与例行」基础设施。
- 新成员尚无 `docs/恢复记忆_<成员名>.md` 或 `shared_rag/private/<成员名>/` 时。

## 输入

- **成员名**（必填）：如 陈正孤、陈小四（与 [项目成员_身份与记忆](../docs/项目成员_身份与记忆.md) 中表格命名风格一致，通常为「陈+称呼」）。
- **角色/职责简述**（可选）：一句话说明身份，如「Tiger 项目新成员，向陈正霞汇报」。
- **继承自**（可选）：要**继承其记忆**的已有角色名，如 陈正霞、陈正落。若指定，则新成员的恢复记忆入口中会加入「先读该角色的近期记忆（及可选本机环境记忆）」一步，以便延续该角色视角的上下文。**默认只继承记忆，不继承身份与职责**（新成员仍有自己的身份、职责与例行工作文档）。若需同时继承身份/职责，见下文「继承记忆 + 身份/职责（可选）」。

## 生成清单（按顺序执行）

### 1. 创建恢复记忆入口

- **文件**：`docs/恢复记忆_<成员名>.md`
- **内容**：参考 [恢复记忆_陈正落](../docs/恢复记忆_陈正落.md)，替换「陈正落」为该成员名、角色描述替换为输入的角色简述；「请按顺序阅读以下文档」中第 2 项改为 `docs/<成员名>_职责与例行工作.md`，第 3、4 项路径改为 `shared_rag/private/<成员名>/本机环境记忆.md` 与 `shared_rag/private/<成员名>/近期记忆.md`。
- **若指定了「继承自」某角色**：在「请按顺序阅读以下文档」中，在本人本机环境记忆/近期记忆**之前**插入一步或若干步：
  - **仅继承记忆（默认）**：阅读 `shared_rag/private/<被继承角色名>/近期记忆.md`（及可选 `本机环境记忆.md`），只读、不写入，延续该角色视角的上下文。
  - **继承记忆 + 身份/职责（可选）**：若希望新角色也按被继承角色的身份与职责行事，额外在恢复记忆顺序中加入：阅读 `docs/<被继承角色名>_职责与例行工作.md`，以及 [项目成员_身份与记忆](项目成员_身份与记忆.md) 中该角色对应小节。此时新角色可视为该角色的「分身/代理」；谁写近期记忆、是否共享同一身份牌等需另行约定（例如约定新角色只读不写被继承角色私有 RAG、本人仍有自己的私有 RAG 与职责文档作为补充）。

### 2. 创建职责与例行工作文档

- **文件**：`docs/<成员名>_职责与例行工作.md`
- **内容**：标题「<成员名> · 职责与例行工作」；身份速览（名字、角色用输入简述）；职责清单可写「待本人补充」；例行工作可写「待本人补充」；**记忆保存约定**一节照抄 [陈正落_职责与例行工作](../docs/陈正落_职责与例行工作.md) 中「五、记忆保存约定」并替换路径为 `shared_rag/private/<成员名>/近期记忆.md`；相关链接含 [恢复记忆_<成员名>](恢复记忆_<成员名>.md)、[项目成员_身份与记忆](项目成员_身份与记忆.md) 等。

### 3. 创建私有 RAG 目录与文件

- **目录**：`shared_rag/private/<成员名>/`
- **文件**：
  - `README.md`：说明「供 cursor_<成员名> 存放仅己用的记忆与上下文」，路径说明，不参与社区检索、不提交仓库。
  - `本机环境记忆.md`：模板，说明「供后续会话恢复上下文用」，本环境配置、凭证与推送等可待本人补充。
  - `近期记忆.md`：占位，可仅含「## 近期记忆（按时间倒序）」，供定时保存记忆用。

### 4. 更新项目成员_身份与记忆

- **文件**：`docs/项目成员_身份与记忆.md`
- **操作**：在「一、成员名单与角色」表格中新增一行（名字、身份/角色、说明）；若有「二、三、四…」按成员分节，则新增一节「N、cursor_<成员名>」；在「五、每人私有 RAG 与例行工作」表格中新增该成员的例行工作文档链接（可为「待本人补充后链入 例行工作_总览」）。

### 5. 更新私有 RAG 总览

- **文件**：`docs/私有RAG_总览.md`、`shared_rag/private/README.md`
- **操作**：在「各成员私有 RAG 路径」表中新增一行：成员名、路径 `shared_rag/private/<成员名>/`、说明（用输入的角色简述或「新成员」）。

### 6. 更新例行工作_总览

- **文件**：`docs/例行工作_总览.md`
- **操作**：在「成员例行工作链接」表中新增一行：cursor_<成员名>、链接到 `docs/<成员名>_职责与例行工作.md`、说明可写「待补充」或角色简述。

### 7. 更新 RAG 副本（若有）

- **文件**：`shared_rag/best_practices/项目成员_身份与记忆.md`
- **操作**：在成员表与相关列表中补充该成员及「恢复记忆_<成员名>」说明，与权威文档保持一致。

### 8. 更新恢复记忆（通用）成员表

- **文件**：`docs/恢复记忆.md`
- **操作**：在「成员名单与入口文档」表中新增一行：成员名、恢复记忆_<成员名>.md、说明（角色简述）。

## 一键脚本（推荐）

在 `agentfuture` 目录下执行：

```bash
python3 scripts/create_member.py <成员名> "<角色/职责简述>"
```

脚本会自动创建：`docs/恢复记忆_<成员名>.md`、`docs/<成员名>_职责与例行工作.md`、`shared_rag/private/<成员名>/README.md`、`本机环境记忆.md`、`近期记忆.md`。**仍需人工**在 `docs/项目成员_身份与记忆.md`、`docs/私有RAG_总览.md`、`shared_rag/private/README.md`、`docs/例行工作_总览.md`、`docs/恢复记忆.md` 中按上面 4～8 步补充该成员（脚本会输出待编辑清单）。
