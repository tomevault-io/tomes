# CLAUDE.md

## Go Error Handling Style

- Keep error variables in the shortest possible scope. Prefer `if err := f(); err != nil` over declaring `err` separately.
- Always use `err` as the variable name for errors.
- When combining multiple errors, use `errors.Join`.

## Documentation Rules

- When modifying the `tobari html` command, always update `docs/cli/html.md` accordingly.

## Temporary File Storage Architecture

tobari uses several temporary file locations for inter-process communication between the cover, compile, and link tool phases. Each location has a specific lifetime and isolation strategy.

**Naming convention:** All tobari-created temp files use the `tobari_` prefix with underscore separators (e.g. `tobari_pkgs.json`, `tobari_main_*.go`). Constants for these names live in `internal/utils/go.go`.

### `$WORK/bNNN/` — Per-package, per-build

Go's build system assigns a unique `$WORK/bNNN/` directory to each package build action. Files here are shared between the cover, compile, and link phases of the same package and are safe from parallel build races.

| File | Writer | Reader | Purpose |
|---|---|---|---|
| `tobari_suppdeps.json` | cover tool | compile tool | Supplementary dependency map for whole-program analysis |
| `covervars.go` | cover tool | compiler | Coverage variables and embedded source code |
| `tobari_pkgs.json` | compile tool | link tool | tobari package export path map |

### `$WORK/tobari_app/` — Per-build, shared across compile/link

A minimal Go module created on demand under Go's `$WORK` directory. Reused across all toolexec invocations of a single `go build`/`go test` command and automatically cleaned up by Go when the build finishes. Created only when needed (main/testmain compile or link cache miss, AND tobari is not already in importcfg).

| File | Purpose |
|---|---|
| `go.mod` | Temporary module requiring the correct tobari version |
| `main.go` | Minimal `import _ "github.com/goccy/tobari"` |
| `go.sum` | Generated by `go mod tidy` |

### `$TMPDIR/tobari/` — Global cache, persistent across builds

Content-hash-keyed caches that survive Go build cache hits (when the cover/compile tool is not re-invoked). Different versions or flags produce different hash keys, so entries are naturally isolated.

| Directory | Key | Purpose |
|---|---|---|
| `cache/` | Runtime export filename (SHA256) | tobari package export paths by build config |
| `coverpkgs/` | Package directory path (SHA256) | Registry of coverage-target packages |
| `overlay/<content_hash>/` | Overlay file content (SHA256, 16 chars) | Modified runtime/testing/testdeps packages |

### Ephemeral temp files (`os.CreateTemp` / `os.MkdirTemp`)

| Pattern | Purpose | Cleanup |
|---|---|---|
| `tobari_main_*.go` | Main hook injection (suppDeps + linkname calls) | OS cleanup |
| `tobari_driver_*.json` | GOPACKAGESDRIVER IPC for SSA analysis | `defer os.Remove` |
| `tobari_coverprofile_*.txt` | Intermediate file for HTML report generation | `defer os.Remove` |
| `tobari_html_*` | Working directory for `tobari html` source extraction | `defer os.RemoveAll` |
